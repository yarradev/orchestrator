**Date:** 2026-07-01 · **Status:** analysis (no code changed) · **Purpose:** capture the *proven* workflow behavior of **yarradev v1** (`yarrasys/yarradev`, plugin `yarradev` v0.3.26, the GitHub-Issues orchestrator battle-tested in **yanyja**) so the new target (single Cloudflare board + `yarradev-board` plugin + `orchestrator-core`) reproduces it. ## 0. The reframe (what parity actually means)

Three facts changed the framing:

1. **The target is the stronger *substrate*.** epoch=`current_gen` w/ typed 409 (vs v1 silent drop); **authenticated capability matrix** (vs v1's grammar-trust + inert `clear_authority`, gh#32); materialized SQL read model + `compile()` **implemented** (vs v1's pinned-summary comment hack + a `yarradev-compile` that was **never built**); proactive lease reclaim via `alarm()` (vs v1's passive session-only reclaim); epic fan-in **primitive already exists**.
2. **v1 is the richer *proven policy*.** content-based `checks/` scanners, ADVICE/HOLD/VETO ladder with re-review-supersedes-VETO, rework-staleness fencing (v0.3.8), board-drift detection, transition-budget that actually bounds respawns.
3. **yanyja tells us which gaps *bite*.** The only sustained real instance ran **Projects-v2, org-federated, two-tier, two advisors, shadow-mode**. It exercised the governance ladder + drift detection + fast-lane for real, and repeatedly got burned by the advisor fail-open, false-production, and missing-releaser bugs. QUESTION/ANSWER + thread_budget were **never exercised at all**.

**Parity = port v1's proven policy onto the target's substrate — ranked by what bit in practice — and do NOT rebuild v1's unfinished vision (`yarradev-compile`, thread_budget were never real).**

## 2. Per-dimension notes (condensed)

**Lifecycle** — v1 story tier design→development→testing→done (+first-class REJECT edges, mechanical/judgement gates); yanyja customized to spec→development→staging→production→done, two lanes, budgets 20/ttl 3600. Target 7-state machine is a superset; gaps are P0-2/P0-3 + epic wiring (P1-5). Methodology §A1 "card is the actor; state on the board; orchestrator memoryless" — target preserves this by construction.

**Protocol/acts** — target act vocabulary is a **superset** of v1's 13 speech-acts (epoch=gen, CONFIG-as-act, materialized reads supersede the pinned summary). Real gaps: ADVICE persistence (P1-2), thread grouping (P2), ESCALATE-as-gate (P2), REJECT `cat:` not structured. Naming crosswalk above.

**Advisors/governance** — v1's strongest, target's weakest. Port: content scanners (P1-9), non-vacuous `advisor_clear` (P0-1), ADVICE/clean persistence for head-fresh re-review (P1-2). Target ahead on enforced `clear_authority` + authenticated identity (P1-10). Generalize the "no-cheapest-model-on-veto" CI guard (yanyja's `legal-advisor: haiku` slipped through).

**Orchestrator loop** — v1 `_decide()` is a deterministic 8-branch precedence function; target's `decide` shrank the vocabulary (13→8→6) and moved enforcement server-side (legit). Fix: respawn backstop (P1-4), malformed/unknown-stage escalate (plugin silently noops), board-drift (P1-3), and reconcile the 3 divergent `decide`s into one (core → generated plugin).

**Import/enforcement** — v1's `yarradev-compile` / constitution chain was **designed, never built**; target's `compile()` + caps already exceed it. Port the *proven* parts: content scanners, a `config_hash` drift check (P1-7), a named-check allowlist (P0-2), and decide whether code-repo CODEOWNERS/branch-protection provisioning (`setup-enforcement.sh`) is in scope (devs still push real PRs to GitHub even though the board left GitHub).
